dg::deepcore::Node Class Referenceabstract
Processing » Framework

Detailed Description

Base class for processing framework nodes.

Note
Node implementations should not inherit from this class directly, use NodeT instead.

Example

class MyNode : public NodeT<MyNode, Node>
{
public:
// Create your input and output Ports, Atributes, and Properties here.
void init() override
{
setInputs({
port::Foo::create("Foo")
});
setOutputs({
port::Bar::create("Bar")
});
setAttrs({
attr::UInt32("Size"),
// ...
});
setProps({
prop::UInt32("ActualSize"),
// ...
});
// do more initialization ...
}
protected:
void setup() override
{
// Do your setup actions here. This is the time to set your Properties.
props("ActualSize").setValue(42);
// ...
}
void tearDown() override
{
// close your files and free your resources here
// ...
}
void pull() override
{
// This is where you request data from your input ports. Usually only
// the sink nodes will do this, but any node is allowed to request
// data.
auto& foo = input("Foo");
foo.put(dataRequest);
}
// You must override this method, the rest are optional
void process() override
{
// do the work here.
auto& foo = input("Foo");
auto& bar = output("Bar");
auto& requests = bar.getAll();
auto& responses = foo.getAll();
for(auto& request : requests) {
// don't forget to check for cancelled() as often as you can
if(cancelled()) { return; }
// .....
foo.put(newRequest);
}
for(auto& response : responses) {
// also a good idea to report progress if you can, that will
// also give you a "cancelled" status as well.
if(!progress(myProgress)) { return; }
// .....
bar.put(newResponse);
}
}
};

virtual void dg::deepcore::Node::process ( )
protectedpure virtual

Performs the main work for this Node.

This method is called when there's data on any of the Node's Ports (input or output). The port can process all the data available on the port, or it can process a single transaction and return. In addition, this method can block for long periods of time, occasionally checking if more data is available on one of the Ports or if processing was cancelled.

Node implementers should check the cancelled() flag as often as it is feasible and use the progress() callback to report progress.

Implemented in dg::deepcore::imagery::GeoBlockSource.

void dg::deepcore::Node::run ( )

Starts processing.

If the node graph is not initialized, this method will initialize the nodes. After all the nodes are initialized, the pull operation will be performed on all nodes in order: from source to sink. This will trigger the nodes to start processing.

If the process is already running this method does nothing.

void dg::deepcore::Node::setup ( )
protectedvirtual

Called from the worker thread before processing starts.

This method is called during initialization. The call will always be made from this Node's worker thread, the same thread as the process() method. Before this method is called, all ports connected to the input Ports of this Node will have been initialized, so their Properties can be accessed.

This method should open files, make connections, and generally allocate resouces. This is also the place to set this Node's Properties.

Reimplemented in dg::deepcore::imagery::GeoBlockSource.

void dg::deepcore::Node::tearDown ( )
protectedvirtual

Cleans up after processing has finished.

This method is called after processing has finished. The call is always made from this Node's worker thread, the same thread as the process() method. When this method is called, other Nodes are also in various stages of shutting down, so you should not access other Nodes from this method.

You should always call Node::tearDown() from your overridden method.

Reimplemented in dg::deepcore::imagery::GeoBlockSource.
