dg::deepcore::ObjectT< T, BaseT > Class Template Reference

Detailed Description

template<class T, class BaseT = Object>
class dg::deepcore::ObjectT< T, BaseT >

Template class used to inherit from Object and classes that inherit from it.

Template Parameters
TName of the child class.
BaseTName of the base class.

Classes inheriting from Object, should use ObjectT instead.

Class initialization should be done in the overridden Object::init() method, not the constructor. Constructors are still allowed, but they will not be able to use the ptr() and wptr() to get the shared pointer to the class'es instance.

Example

// Class inherits from Object
class A : public ObjectT<A>
{
A();
A(const char* one, int two, double three);
// ................................
protected:
void init() override
{
// most of the class initialization goes here instead of the constructor.
}
}
// You should always register your Object descendants with the Type system
namespace types { const TypeT<char> A("A"); }
// Class inherits from an Object ancestor
class B : public ObjectT<B, A>
{
// ...............................
}
// No need to specify base class when registering with the type system.
namespace type { const TypeT<B> B("B"); }

Instances of the inheriting classes should never be created using the
new operator. You must always use the create() factory method.

Example

// a's type is std::shared_ptr<A>, or A::Ptr for short.
auto a = A::create();

Classes inheriting from Object that are expected to be created and that are not just abstract base classes must be default-constructible. However, it is possible to have parametrized constructors for convenience. Classes must allow initializing of their state without using parametrized constructors.

Example

// Will use an A::A(const char* one, int two, double three) constructor.
auto a = A::create("one", 2, 3.0);
